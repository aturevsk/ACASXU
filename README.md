# ACAS Xu — ONNX to Quantized Simulink C Code Generation

Import an ACAS Xu (Airborne Collision Avoidance System) ONNX neural network into MATLAB, **quantize to INT8**, export to Simulink via `exportNetworkToSimulink`, and generate embedded C code via Embedded Coder.

## Model

| Output | 5 advisory scores (COC, Weak L/R, Strong L/R) |
| Quantization | INT8 (weights + activations) |

## Pipeline

| Step | Method | Result |
|------|--------|--------|
| 1 | `importONNXNetwork` / `importNetworkFromONNX` | Custom autogenerated layers (incompatible) |
| 2 | Rebuild as native `dlnetwork` | 14-layer network, all standard MATLAB layers |
| 3 | Numerical verification | Max error 2.21e-04, cosine similarity 1.0 |
| 4 | `dlquantizer` + `quantize` | INT8 quantized network (28 calibration entries) |
| 5 | `exportNetworkToSimulink` | Simulink model with FC/ReLU subsystem blocks |
| 6 | Embedded Coder (`ert.tlc`) | C code: 320 lines + 306 KB quantized weights |

### Key Finding

`importONNXNetwork` produces custom autogenerated layers (`CustomInputLayerMultiOutput`, fused `MatMul_To_AddLayer1000`) that are incompatible with `dlquantizer` and `exportNetworkToSimulink`. The solution is to **rebuild the network** using native MATLAB layers (`fullyConnectedLayer` + `reluLayer`), transfer the weights, and proceed with the standard quantization/export/codegen workflow.

## Generated C Code

The quantized Simulink model generates embedded C with:
- **int8_T** weights and hidden activations
- **int32_T** bias accumulators
- **real32_T** final output (5 advisory scores)
- Bit-shift quantization scaling (>> 6, >> 3, >> 4)
- Real-time execution harness (`rt_OneStep`)
