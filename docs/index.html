1

ONNX Import Attempt FAILED

Both importONNXNetwork and importNetworkFromONNX produced custom autogenerated layers (e.g., gemm operations mapped to custom MATLAB classes). These custom layers are not supported by exportNetworkToSimulink or Embedded Coder, blocking the deployment path.

2

Native Rebuild SUCCESS

Rebuilt the network as a 14-layer dlnetwork using only native MATLAB layers: featureInputLayer, fullyConnectedLayer, and reluLayer. Weights and biases were extracted from the ONNX import and transferred to the native layers.

3

Verification PASSED

Compared outputs of the ONNX-imported network vs. the native rebuild across test inputs. Maximum absolute error: 2.21e-04. Cosine similarity: 1.0. The small numerical differences are due to floating-point operation ordering.

4

INT8 Quantization INT8

Applied post-training quantization using dlquantizer with 1,000 calibration samples. Weights quantized to int8_T, biases to int32_T, with per-layer scale factors implemented as bit-shift operations in the generated C code.

5

Simulink Export SIMULINK

Exported via exportNetworkToSimulink which creates a Simulink model with DataTypeConversion blocks at input and output boundaries to handle INT8 casting. Each fully connected layer becomes a Simulink subsystem block.

6

C Code Generation EMBEDDED CODER

Generated C code using Embedded Coder with ert.tlc system target file. InstructionSetExtensions set to 'None' for maximum portability. Produces 320 lines of C in the main step function with quantized int8 computations.

Quantization Notes

Bit-shift scaling replaces floating-point multiplication with integer shift operations for efficiency on embedded targets. For example, >> 6 divides the accumulator by 64, implementing the combined weight/activation scale factor. The final layer (fc7) outputs real32_T (float) because the ACAS Xu advisory scores require floating-point comparison.

Quantized Computation Flow (Step Function)

The generated C step function implements INT8 matrix multiplication with bit-shift scaling between layers:

// 1. Cast float inputs to int8
input_int8 = (int8_T)roundf(input_float * input_scale);

// 2. INT8 matrix multiply + INT32 bias add
for each layer fc_k:
  acc_int32 = bias_int32[j];
  for i = 0..N:
    acc_int32 += weight_int8[j][i] * input_int8[i];

// 3. Bit-shift scale + ReLU
  output_int8 = (int8_T)max(0, acc_int32 >> shift_k);

// 4. Final layer: cast int32 accumulator back to float
output_float = (real32_T)acc_int32 * output_scale;
